A problem-first learning rail. Where the guides are recipes, the tutorials build
understanding from a concrete starting point.

The rail is also the last leg of a larger arc. The
[keelson tutorials](https://danielscholl.github.io/keelson/docs/tutorials/) end by
building a real app from a template, and the
[Chamber rail](https://danielscholl.github.io/keelson-rib-chamber/tutorials/) puts
rooms on its plan and its code. This rail staffs it: the team you cast scans that
same project, and the work you hand it comes off the project's own
[backlog](https://github.com/danielscholl/keelson-sample/blob/main/backlog.md).
Any real repository works in its place, every page says what it needs, but riding
the arc in order means one app carries you from first build to a standing team
that maintains it.
