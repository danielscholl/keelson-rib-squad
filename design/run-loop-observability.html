Watching the minds work

The Run loop isn't failing at layout — it's failing at evidence. The board summarizes a run the operator can't inspect, because the richest signal (tool calls, thinking, tokens) is discarded at capture and the rest is truncated at render. The redesign below reorganizes the board around the run's real unit — the round — and gives every mind a legible lane. Most of it needs no host change.

What the console loses today

I drove the live surface with Playwright, read the r11 run's stored ledger, and traced the pipeline through the rib source. The gap is two-layered — capture first, presentation second:

  • discarded

    The turn stream is drained and thrown away. Every tool_use, tool_result, and thinking chunk the provider emits crosses the seam and dies in an empty loop. This is the single root cause of "I can't see what the minds are doing."turn-runner.ts:86 — for await (const _chunk of turn.stream) {}

  • discarded

    Per-turn token usage is never captured, though RibAgentTurnResult.usage already carries it since the usage seam shipped. Zero grep hits for usage in squad src.feeds #50 (cost visibility) for free

  • truncated

    Ledger entries cap at ~1,500 chars — the member's inter-tool narration jammed into one run-on string. No timestamps, no duration, no per-entry structure beyond round / kind / speaker / text.coordinator.ts — CoordinatorEntry

  • truncated

    The board then shows the last 12 entries at 200 chars, flat. The r11 run has 29 entries — rounds 0–7 simply aren't on the board, and nothing on the board expands. Findings, standup, and goal are cut mid-sentence with no path to the rest.boards/coordinator.ts — MAX_TRANSCRIPT=12, TRANSCRIPT_CAP=200

  • broken

    The per-step repo footprint is wrong in practice: every code entry in r11 says 1 file, +0/−0 — the diff capture surfaced an untracked spec file instead of the real change (the #108 bug, visible in stored data).runs/2026-07-02T16-14-45.json

  • confusing

    Member identity hues are hashed from IDENTITY_TONES = [accent, info, caution, ok, brand] — a member can hash to caution orange or ok green, so identity masquerades as status. Edie currently renders warning-orange on every row. Raw markdown (##, **, backticks) leaks into transcript rows and roster charters.boards/coordinator.ts — identityTone()

1Cost strip

Round is a meter, not a number — proximity to the budget ceiling is the message, and a track shows it preattentively (tone flips to caution at 80%). Tokens and elapsed ride the same strip. Dashed-underline values need the capture tier.

Tier A capture · feeds #50, #72

2Round rail

The run's shape at a glance: who acted each round, and where it went red. R0–R6 grind, R7 review, R8 gate-red, R9 fixing — the story the flat transcript hides. Clicking a round filters the ledger. Member hue = identity; red is reserved for the gate. Renderable today with the grid canvas primitive (toned badges per cell).

Tier B board · rib-only

3Now card — the live mind

The in-flight turn shows its streaming tool trace (last 3 calls, verb + target + outcome), elapsed, and token tally — republished on a ~2s throttle as chunks arrive, so the existing live-dot (the #353 pulse) finally has something live behind it. This is the single biggest "what is it doing?" win.

Tier A capture + Tier B board · fixes the heart of #72

4Round-grouped, expandable ledger

Entries group under a round head with a phase caption, newest first; every row expands to the full stored text (1,500 chars, markdown stripped at compose). Kind glyph = status; chip = identity — separate channels. Gate events render at error weight instead of hiding among findings. Expansion needs one host primitive: a detail? field on rows items.

Tier C host (small) + Tier B board · fixes #111's invisibility too

5Collapse the grind, keep the tail

Old rounds compress to a one-line stub instead of being silently dropped — the current board hides 17 of r11's 29 entries with no trace. History stays reachable; the fresh rounds get the space.

Tier B board · rib-only

6Minds panel — identity done right

One lane per member: role, provider · model, turns, tokens, footprint, and what they're doing now. Identity hues come from a validated pair (cyan/amber — CVD ΔE 73+, both themes) instead of hashing into the status ramp where edie renders as a warning. "Worked by" generalizes into this panel.

Tier B board · rib-only (hue pool fix); optional Tier C: first-class identity tones in canvas

What it takes

Ordered so each tier ships alone and pays for itself. Tier A is the keystone — without capture, the board redesign polishes a view of data that doesn't exist.

What I deliberately didn't propose

Streaming raw thinking token-by-token into the board (the snapshot substrate is frames, not a chat stream — the 2s-throttled trace gets 90% of the value); a chat-style transcript surface per member (the Enter Edie seeded-chat path already exists for interrogation); and any new top-level view — this is a recomposition of one board plus one host primitive, not a new surface.

Evidence: Playwright drive of the live surface at :7878 (squad + chamber + chat panels), the stored r11 ledger (29 entries), and the rib + canvas source. No coordinate run was fired — the stored ledger and code paths carried the same evidence without the cost. Identity palette validated both themes (light #0891b2/#b45309 · dark #0891b2/#d97706, CVD ΔE ≥ 73, contrast ≥ 3:1). Chat's ToolCallsBlock (tool name + owning-rib chip, expandable) is the in-house precedent the Now card follows.
